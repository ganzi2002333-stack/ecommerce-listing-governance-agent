## 2. 项目背景

在跨境电商平台中，商家每天会上传大量商品 listing，包括商品标题、描述、品牌、类目、价格、物流承诺等信息。由于不同国家和地区存在法律法规、平台政策、商品治理标准和语言表达差异，商品信息中可能出现夸大宣传、禁限售商品、假货风险、医疗功效声明、隐私安全风险、危险品风险、认证信息不实、商品描述缺失等问题。

传统商品治理主要依赖人工审核和规则系统。人工审核成本较高、响应速度有限；纯规则系统虽然稳定，但对复杂文本语义、商家表达差异和多场景解释能力支持不足。因此，需要探索 AI Agent 在商品治理和商家服务中的应用，通过“规则检索 + 风险判断 + 整改建议 + 智能问答”的方式提升平台治理效率和商家体验。

本项目基于公开电商商品数据与自建商品治理规则库，设计一个面向跨境电商场景的 AI Agent 原型，用于验证 AI 在商品治理与商家服务中的可行性。

## 3. 产品目标

### 3.1 业务目标

1. 提升商品 listing 风险识别效率，减少人工初筛成本。
2. 帮助运营和治理人员快速定位商品风险类型和命中规则。
3. 为商家提供清晰、可执行的整改建议，降低商家理解成本。
4. 探索 AI Agent 在智能客服和商品治理场景中的落地方式。
5. 通过小规模测试集评估 Agent 风险识别效果，为后续迭代提供依据。

### 3.2 用户目标

| 用户角色   | 用户目标                 |
| ------ | -------------------- |
| 平台运营人员 | 快速了解商品存在的问题，并跟踪风险分布  |
| 商品治理人员 | 判断商品是否需要人工复核，并查看命中规则 |
| 商家服务人员 | 快速解释商品被拦截或提示修改的原因    |
| 平台商家   | 理解商品问题，并获得可执行的修改建议   |

## 4. 用户痛点

### 4.1 平台侧痛点

1. 商品 listing 数量大，人工审核成本高。
2. 规则复杂，治理人员需要频繁查阅政策文档。
3. 不同风险类型之间边界不清，容易出现误判或漏判。
4. 普通规则系统难以解释“为什么判定为风险”。
5. 风险识别结果难以直接转化为商家可理解的整改建议。

### 4.2 商家侧痛点

1. 商家不清楚商品为什么被判定为风险。
2. 商家不了解具体命中的治理规则。
3. 商家不知道标题或描述应该如何修改。
4. 多次修改仍无法通过审核，影响商品上架效率。
5. 对平台治理规则的理解成本较高。

---

## 5. 产品定位

本产品定位为：

> 面向跨境电商平台商品治理与商家服务场景的 AI Agent 原型，支持商品 listing 风险识别、治理规则检索、风险原因解释和商家整改建议生成。

本产品不是替代人工审核，而是作为 AI 辅助工具，为平台提供初筛、解释和建议能力。

---

## 6. 核心使用场景

### 场景一：单条商品风险检测

运营或治理人员输入单条商品信息，系统自动判断风险等级、风险类型、命中规则，并生成整改建议。

### 场景二：商家服务辅助答疑

商家询问“为什么我的商品被判定为风险”“应该如何修改标题或描述”，系统基于风险判断结果和规则库生成解释。

### 场景三：批量商品检测

运营人员上传 CSV 文件，系统批量分析商品风险，并输出风险等级、命中规则、是否需要人工复核和整改建议。

### 场景四：效果评估与复盘

通过人工标注测试样本，对 Agent 判断结果进行准确率、召回率、混淆矩阵和错误样本分析，为后续优化提供依据。

## 8. 用户流程

### 8.1 单条商品检测流程

```text
用户输入商品信息
        ↓
系统解析商品标题、描述、品牌、类目
        ↓
系统检索相关商品治理规则
        ↓

        ↓
系统展示命中规则、判断依据和风险解释

        ↓
```

```text

        ↓
生成批量检测结果表
        ↓

        ↓

```

## 9. 功能需求说明

### 9.1 商品信息输入模块

#### 功能描述

用户可以输入商品的基础 listing 信息，包括：

* Product ID
* Product Title
* Brand
* Category
* Product Description

#### 输入字段

| 字段          | 类型     | 是否必填 | 说明     |
| ----------- | ------ | ---- | ------ |
| product_id  | string | 否    | 商品唯一标识 |
| title       | string | 是    | 商品标题   |
| description | string | 是    | 商品描述   |
| brand       | string | 否    | 商品品牌   |
| category    | string | 否    | 商品类目   |

#### 验收标准

* 用户可以正常输入商品信息；
* 标题和描述为空时，应提示用户补充；
* 输入完成后可以触发 Agent 检测。

---

### 9.2 商品治理规则检索模块

#### 功能描述

系统将商品标题、描述、品牌和类目拼接为查询文本，通过向量检索从规则库中召回最相关的商品治理规则。

#### 规则来源

规则库基于公开电商平台商品治理场景进行人工归纳，存储在：

```text
data/rules.json
```

规则覆盖：

* 误导性声明；
* 医疗功效声明；
* 禁限售商品；
* 假货与知识产权风险；
* 隐私安全风险；
* 危险品风险；
* 商品质量问题；
* 认证声明风险；
* 物流服务承诺风险。

#### 验收标准

* 系统能够返回相关规则；
* 每条规则包含 rule_id、rule_name、risk_type、risk_level、description 等字段；
* 高风险关键词商品能够召回对应规则，例如 replica、gun、cure、hidden camera 等。

---

### 9.3 风险判断模块

#### 功能描述

AI Agent 基于商品信息和召回的规则，判断商品 listing 的风险等级、风险类型、命中规则和判断依据。

#### 输出字段

| 字段                 | 类型      | 说明                  |
| ------------------ | ------- | ------------------- |
| risk_level         | string  | High / Medium / Low |
| risk_types         | list    | 风险类型                |
| matched_rule_ids   | list    | 命中的规则 ID            |
| evidence           | list    | 商品文本中的风险证据          |
| explanation        | string  | 风险判断解释              |
| needs_human_review | boolean | 是否建议人工复核            |

#### 风险等级定义

| 风险等级   | 判断标准                                    |
| ------ | --------------------------------------- |
| High   | 涉及禁售、受限商品、武器、药品、假货、隐私侵犯、危险品、金融欺诈等明显高风险  |
| Medium | 涉及夸大宣传、认证声明不清、误导性促销、类目不匹配、可疑 IP 使用等中等风险 |
| Low    | 普通商品、轻微信息缺失、品牌不清楚、描述较短但无明显安全或合规风险       |

#### 验收标准

* Agent 能输出结构化 JSON；
* 输出中包含风险等级、风险类型、命中规则和解释；
* 对明显高风险商品能够判断为 High；
* 对正常商品应尽量避免误判为 Medium 或 High。

---

### 9.4 商家整改建议模块

#### 功能描述

系统根据风险判断结果，生成面向商家的整改建议，帮助商家理解问题并修改商品信息。

#### 输出字段

| 字段                    | 类型     | 说明       |
| --------------------- | ------ | -------- |
| merchant_message      | string | 面向商家的解释  |
| action_items          | list   | 具体整改动作   |
| rewritten_title       | string | 修改后的标题建议 |
| rewritten_description | string | 修改后的描述建议 |

#### 示例

原商品标题：

```text
Whitening Anti-aging Face Cream 100% Effective
```

整改建议：

```text
Moisturizing Face Cream for Daily Skincare
```

#### 验收标准

* 建议语言清晰、可执行；
* 能指出需要删除或修改的风险表达；
* 能生成更安全的标题和描述；
* 不应生成新的夸大宣传或违规表述。

---

### 9.5 批量 CSV 检测模块

#### 功能描述

用户上传 CSV 文件后，系统批量检测多条商品 listing，并输出风险分析结果。

#### 输入格式

CSV 文件建议包含以下字段：

```text
product_id
title
description
brand
category
```

#### 输出结果

| 字段                 | 说明       |
| ------------------ | -------- |
| product_id         | 商品 ID    |
| title              | 商品标题     |
| risk_level         | 风险等级     |
| risk_types         | 风险类型     |
| matched_rule_ids   | 命中规则     |
| needs_human_review | 是否需要人工复核 |
| explanation        | 风险解释     |
| merchant_message   | 商家提示     |
| rewritten_title    | 修改后的标题建议 |
| elapsed_seconds    | 单条处理耗时   |

#### 验收标准

* 支持上传 CSV 文件；
* 支持限制处理条数；
* 能生成批量检测结果表；
* 支持下载结果 CSV；
* 能展示风险等级分布图。

---

### 9.6 效果评估模块

#### 功能描述

通过人工构造并标注的测试样本，评估 Agent 风险识别效果。

#### 测试集

```text
data/risky_test_cases.csv
```

#### 评估指标

* Accuracy；
* Precision；
* Recall；
* F1-score；
* Confusion Matrix；
* Average response time；
* Incorrect case analysis。

#### 当前评估结果

```text
Accuracy: 0.80
High-risk recall: 100%
Medium-risk recall: 100%
Low-risk recall: 20%
```

#### 结论

当前 Agent 对高风险和中风险样本识别较好，但对 Low-risk 样本存在误报，部分正常商品被判断为 Medium-risk。后续需要通过补充正常样本、优化 Prompt 和细化风险等级规则降低误报率。

## 11. 指标体系

| ------- | ---------------- |

| 整改建议采纳率 | 商家是否采用系统建议修改商品   |

## 12. 异常场景与处理

| 异常场景           | 处理方式                     |
| -------------- | ------------------------ |
| 商品标题为空         | 提示用户补充标题                 |
| 商品描述为空         | 提示用户补充描述                 |
| CSV 缺少必要字段     | 提示字段格式要求                 |
| Agent 输出非 JSON | 尝试解析 JSON；失败则返回原始输出并提示异常 |
| Ollama 未启动     | 提示用户检查本地模型服务             |
| 规则库为空          | 提示用户先运行 build_kb.py      |
| 处理时间过长         | 建议减少批量检测条数               |

## 13. 风险与限制

1. 当前规则库规模较小，仅覆盖常见商品治理场景。
2. 测试集样本数量较少，评估结果仅代表小规模验证。
3. 当前版本主要处理文本信息，尚未接入商品图片审核。
4. 本地模型推理速度较慢，不适合大规模实时处理。
5. Agent 对 Low-risk 商品存在一定误报。
6. 当前版本尚未接入真实商家后台、客服系统和人工复核流程。
7. 输出结果仅作为辅助判断，不应直接替代平台最终审核决策。

## 14. 后续迭代方向

### 14.1 短期优化

* 补充更多正常商品样本；
* 优化风险等级 Prompt；
* 细化 Low-risk 与 Medium-risk 的判断边界；
* 增加快速评估模式，仅评估风险等级，不生成整改建议；
* 优化 Streamlit 页面展示效果。

### 14.2 中期优化

* 增加商家客服问答模块；
* 支持商家追问“为什么被判风险”和“如何修改”；
* 增加人工复核入口；
* 支持多语言商品 listing 审核；
* 增加更细粒度的风险标签。

### 14.3 长期优化

* 接入商品图片审核能力；
* 接入真实平台工单和客服系统；
* 建立商家申诉和反馈闭环；
* 根据真实审核结果持续优化规则库和 Prompt；
* 构建治理数据看板，展示风险趋势、误报率和复核结果。

## 15. MVP 验收标准

本项目 MVP 验收标准如下：

1. 可以输入单条商品信息并完成风险判断；
2. 可以检索并展示相关商品治理规则；
3. 可以输出结构化 JSON 风险判断结果；
4. 可以生成商家可理解的整改建议；
5. 可以上传 CSV 并完成批量检测；
6. 可以下载批量检测结果；
7. 可以基于人工标注测试集输出评估指标；
8. README、PRD 和截图完整，能够支持面试展示。

## 16. 总结

本产品原型验证了 AI Agent 在跨境电商商品治理和商家服务场景中的基本可行性。通过规则知识库、RAG 检索、风险判断和整改建议生成，系统能够辅助平台完成商品风险识别和商家解释工作。

当前版本已经实现 MVP 核心链路，但仍需要在规则覆盖度、低风险误报、响应速度、多语言能力和人工复核闭环方面继续优化。未来可进一步扩展为面向平台运营、商品治理和商家客服的一体化 AI 辅助工具。
